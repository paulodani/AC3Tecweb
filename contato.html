SAIBA SOBRE!!

Bootstrap 3.4.1 e 4.3.1 13 Fevereiro 2019


No início desta semana, um desenvolvedor relatou um problema de XSS semelhante à vulnerabilidade de destino de dados corrigida na v4.1.2 e v3.4.0: o atributo de modelo de dados para nossos plug-ins pop-up e de dica de ferramenta não tinha sanitização XSS adequada do HTML no valor do atributo.


Para resolver o problema, implementamos um novo sanitizer JavaScript para permitir apenas elementos HTML na lista de permissões no atributo de dados. Você pode modificar nossa implementação de sanitização para personalizar a lista de permissões do elemento HTML, desabilitar totalmente a limpeza ou passar sua própria função de higienização (útil se você usar sua própria biblioteca). No entanto, para proteção adicional, não há como modificar a higienização por meio de atributos de dados - você deve modificar essas opções de plug-in por meio da API JavaScript.


Aqueles que modificaram os modelos padrão, leia os novos documentos do v4.3 sanitizer ou os novos documentos do v3.4 sanitizer.


À luz desta vulnerabilidade, também estamos a auditar os nossos fluxos de trabalho de relatórios de segurança para garantir que estão atualizados. Isso incluirá etapas como adicionar um arquivo SECURITY.md ao nosso repositório e garantir que nossos canais e processos privados estejam atualizados e documentados com a equipe.


Obrigado a poiu por relatar a vulnerabilidade ao projeto Bootstrap Drupal e a Mark Carver do projeto Bootstrap Drupal por nos revelar o problema com responsabilidade. Também um grande obrigado a @ Johann-S, @Xhmikosr e @bardiharborow em nossa equipe pela rápida entrega dos lançamentos de hoje.
